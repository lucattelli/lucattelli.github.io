Extending a Resource File in UI5

When dealing with a simple extension project in Fiori, it’s common to add new fields or labels to a view. It’s a good idea to keep your own resources file in the extension project, so that you keep relying on UI5 ability to detect the language and read the corret texts to build the view.

Since its a fragment view in an extension project, all the effort to declare and load a resource model is done by the standard app. We only need to enhance the model with our own text elements.

To do so, apply these two lines in your extension controller.

	var oResModel = this.getView().getModel("i18n");
	oResModel.enhance({bundleUrl:"./i18n/myresourcefile.properties"});

It will automatically search for language-dependent versions of the file, like myresourcefile_en_US.properties and myresourcefile_en.properties before it reads the default myresourcefile.properties one.

Very simple. Very useful.
